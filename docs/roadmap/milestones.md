# Milestones — Out-door

Milestones are binary: done or not done.
Each milestone has one owner, one success condition, and a target date.

---

## Active Milestones (Phase 0)

### M0.1 — Legal Entity Live
**Owner:** Founder
**Success condition:** Company registered, bank account open, domain secured
**Target:** Week 2

### M0.2 — Hardware Selected
**Owner:** Founder / Ops
**Success condition:** Android tablet model chosen, supplier identified, unit cost locked
**Target:** Week 3

### M0.3 — Dashboard Scoped
**Owner:** Founder / Tech
**Success condition:** Dashboard requirements defined, tech stack chosen, development started
**Target:** Week 4

### M0.4 — First Operator Conversation
**Owner:** Founder

**Target:** Week 5

**Owner:** Founder

## Milestone Tracking Rule

A milestone is only complete when the success condition is verifiably met —
not when work is in progress, not when it is "almost done."

Review milestones weekly. Mark completed milestones with date and brief note.
